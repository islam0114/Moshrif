Moshrif University ERP - System Architecture
============================================

يتكون المشروع من 4 مكونات (Micro-services) رئيسية تعمل بشكل متكامل:

1. محرك الذكاء الاصطناعي (01_AI_Engine) - [AI & Computer Vision Layer]:
   - اللغة: Python
   - التقنيات البارزة: OpenCV, dlib (face_recognition), Flask
   - الوظيفة:
     * يلتقط الفيديو المباشر من الكاميرا.
     * يتعرف على الوجوه ويقوم بعمل Liveness Check (كشف التلاعب عبر حركة ورمشة العين Eye Aspect Ratio) للتأكد أنه شخص حقيقي وليس صورة ممسوكة أو موبايل.
     * يتأكد من أن الطالب ينظر للكاميرا بشكل مباشر.
     * يرسل طلب POST آمن (Authenticated via JWT) للـ Backend بمجرد التحقق.
     * يبث الفيديو (Stream) بدون واجهة رسومية مباشرة إلى الـ Frontend لمراقبته.

2. الخادم الخلفي (02_Backend_API) - [Business Logic Layer]:
   - اللغة: Python
   - التقنيات البارزة: FastAPI, SQLite, Pandas, JWT Auth, smtplib
   - الوظيفة:
     * هو حلقة الوصل بين الذكاء الاصطناعي، وقاعدة البيانات، وواجهة المستخدم.
     * يوفر API endpoints آمنة (محمية بـ JWT).
     * يحتوي على نظام إرسال إيميلات HTML احترافية للطلبة فور تسجيل الحضور (عبر SMTP).
     * يعالج الإحصائيات (Dashboards Stats) باستخدام مكتبة Pandas، مثل استخراج التقارير ونسب الحضور والغياب للطلاب ورفعها في ملفات مساعدة (Excel).

3. قاعدة البيانات (03_Database) - [Data Layer]:
   - التقنية: SQLite
   - الوظيفة: قاعدة بيانات علائقية (Relational Database) بسيطة وخفيفة تحتفظ بكل بيانات النظام.
   - الجداول الرئيسية:
     * students (الطلاب)
     * courses (المواد الدراسية)
     * lecture_schedule (جدول المحاضرات والقاعات)
     * registrations (تسجيلات الطلاب في المواد)
     * attendance_log (سجلات وتواريخ الحضور والغياب)

4. واجهة المستخدم (04_Frontend_Web) - [Presentation Layer]:
   - التقنية: React.js
   - التصميم: TailwindCSS, Recharts
   - الوظيفة:
     * لوحة تحكم ذكية (Smart Dashboard) تعرض الإحصائيات.
     * شاشة استعراض الفيديو المباشر القادم من AI_Engine لعرض الحضور بشكل حي (Live Monitor).
     * لوحة تحكم للمشرفين (Admin Panel) لإضافة وتعديل الطلاب والجداول والمواد.
     * واجهة سلسة وسريعة تعتمد على الـ Components الخاصة بـ React.

--------------------------------------------
الية تدفق البيانات (Data Flow):
1. الكاميرا تلتقط وجه الطالب المار في القاعة.
2. الـ AI Engine يحلل الوجه، يفحص الـ Liveness، ويرسل الـ ID للـ Backend API.
3. الـ Backend يتحقق من جدول القاعة (هل توجد محاضرة الآن؟).
4. يسجل الحضور في 03_Database (في جدول attendance_log).
5. يرسل الـ Backend فوراً إيميل تأكيد للطالب بنجاح العملية.
6. الـ Frontend يقوم بتحديث الـ Dashboard وعرض صورة واسم وحالة الطالب بشكل مباشر دون الحاجة لعمل Refresh.
